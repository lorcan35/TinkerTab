TinkerClaw Mode: 6 Approaches

All designed within ESP32-P4 constraints: 55 LVGL objects, 720x1280 portrait, voice-first, hide/show pattern

A

Enhanced Chat (Inline Agent Layer)

Same chat screen — but when TinkerClaw is active, tool executions appear as special bubbles ("Searching the web...", "Recalled 3 memories"), suggestions become agent-aware, and a subtle pulsing indicator shows agent state. Zero new screens.

Objects: +5 | Files: ui_chat.c only | Risk: Low
B

Agent Drawer (Pull-up Panel)

Chat stays as-is. A small handle at the bottom of the chat screen (above the input bar) can be tapped to slide up a half-screen panel showing: active tool status, last 3 memory facts, quick skill buttons. Dismisses with swipe-down. Think iOS Control Center — always one tap away, never in the way.

Objects: +10-12 (hidden when collapsed) | Files: new ui_agent_drawer.c | Risk: Low-Med
C

Mode Morphing (Screens Transform)

No new screen. When TinkerClaw is active: Home orb area shows last agent action + tool count. Chat header shows agent model + memory indicator. Settings gets an "Agent" section with capabilities list. The whole device subtly changes personality.

Objects: +8 (spread across screens) | Files: ui_home.c, ui_chat.c, ui_settings.c | Risk: Medium
D

Voice Overlay Evolution

The voice overlay (orb screen) becomes the agent cockpit. When you tap the orb in TinkerClaw mode, instead of just the listening animation, you see: live tool execution ("Searching..."), streaming response text, and a context ring around the orb showing memory/tool activity. Chat stays for history. The VOICE experience is where the agent lives.

Objects: +6-8 (reuses voice overlay) | Files: ui_voice.c | Risk: Low — voice overlay already exists
E

Notification Stream (Agent Feed)

A lightweight scrolling feed that replaces the Chat home/welcome view when in TinkerClaw mode. Shows timestamped agent events: "Memory stored", "Web search: 3 results", "Tool: calculator", "Context: 23/50 messages". Tap any event to see details. Like a system log but beautiful. Conversation stays in chat bubbles above.

Objects: +8-10 (recycled list) | Files: ui_chat.c (mode-conditional) | Risk: Low-Med
F

Hybrid: A + D (Chat + Voice Agent)

Best of both worlds. Chat screen gets inline tool bubbles and agent suggestions (approach A). Voice overlay gets the agent cockpit treatment with live tool status around the orb (approach D). Together they cover both interaction modes — text conversations show agent activity inline, voice interactions show it visually around the orb. No new screens, no drawers, just the two existing interfaces made agent-aware.

Objects: +10-12 total | Files: ui_chat.c + ui_voice.c | Risk: Low — both files already exist

My Recommendation: F (Hybrid)

TinkerTab is voice-first — the orb IS the product. Making the voice overlay agent-aware (D) serves the primary interaction. Adding inline tool bubbles in chat (A) covers the text path. No new files, no new navigation, no object budget risk. Both Chat and Voice already use the hide/show pattern. You get agent visibility everywhere the user actually interacts.

You can also combine approaches — click multiple if you like. These aren't mutually exclusive.
