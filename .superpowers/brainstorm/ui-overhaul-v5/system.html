Every screen, every state.

One design language applied to the whole device. No more card-stack pretending to be a phone. The orb is the protagonist; typography and hairlines carry hierarchy; the keyboard earns its space. This doc is the spec.

One object, many meanings.

02c · FLAT DENSITY

Flat — the cheapest.

Same composition, zero fancy rendering. Orb is a single lv_obj circle with a 2-stop linear gradient (native LVGL — no pre-baked image, no radial). No halo, no concentric rings, no inner pulse, no specular highlight. Only animation is a ±3 px Y float on a 6 s ease curve — cheaper than the clock ticking. Runs comfortably on ESP32-S3 / C3 / ESP32 plain, let alone the P4.

What runs smoothly on ESP32-P4 @ 360 MHz · LVGL 36 FPS · PARTIAL render · 144 KB draw buffers

▸ CHEAP — native LVGL, no special tricks
· Solid-fill rectangles & circles (any size)
· 2-stop LINEAR gradient (bg_color → bg_grad_color)
· Borders, rounded corners, opacity
· Text labels (Montserrat) at any size
· Single-color shadow_width ≤ 20 (small glow under the orb)
· lv_anim on x/y/width/opa/rotate
· Hairline rects (1 px separators)
· Partial-redraw regions when only the orb bobs (~2000 px/frame)

▸ FEASIBLE BUT NEEDS CARE
· Radial gradient on the orb — LVGL doesn't support radial natively. Must pre-bake a 512×512 PNG (decoded once into PSRAM ≈ 512 KB) and render as lv_image. One-time cost, zero per-frame work.
· Halo — same trick (pre-baked 720×720 amber PNG with transparent edges).
· Inner pulse / specular — pre-baked OR skip entirely (they're the smallest visual win for the biggest cost).
· Big soft shadow (shadow_width > 30) — measurable frame-time hit during animation. Use small shadow + pre-baked halo image instead.

▸ AVOID
· Live radial gradients (not supported — would need lv_canvas pixel writes per frame)
· 3 concentric ripples animating simultaneously on the Listening state — stagger to 1 at a time or skip on FLAT density
· Blur / backdrop-filter / text-shadow (LVGL can't)
· Animating width/height on large objects (triggers full redraw); animate transform instead
· Simultaneous animations on more than 2 elements per view (our own rule — excessive-motion)

Honest verdict per density

FLAT · always fast
All native LVGL primitives. 36+ FPS guaranteed with 20 ms/frame headroom to spare. Runs on lesser chips too — S3, C3, plain ESP32.
MINIMAL · 3 pre-baked images
Orb face + halo + specular decoded from PNG at boot (~1.2 MB PSRAM). Float + halo-breath only. Holds 36 FPS on P4, confirmed by the current build's counter. Small device risk: none.
HARDCORE · same as Minimal
Extra telemetry strips are just text labels with periodic updates (10 Hz). Negligible cost. The mini RMS graph is 12 thin rects with width/opacity anim — cheap. Stays at 36 FPS.

Voice states — where we spend the budget

Listening ripples — 3 concentric circles growing + fading simultaneously. On FLAT density we show 1 ripple at a time (staggered 800 ms). On Minimal/Hardcore, the full 3-ripple effect with a pre-baked halo pulse.
Processing rotation — lv_anim on transform_angle of a pre-baked ring image. Cheap.
Speaking TTS-sync — animate shadow_opa 160 ↔ 220 at the mic RMS rate. One property, one object. Cheap even on P4.
Quiet hours — static. Just lv_obj_set_style_opa(orb, 140, 0). Free.

04 · CHAT

A letter, not a card stack.

AI messages have no bubble — they're indented body text behind a thin amber rule, so the conversation reads like a letter. User messages are the only "card" — amber pill, black text. Tool calls are hairline-bordered paragraphs, not widgets.

APPENDIX · PRINCIPLES

Rules we keep.

Hierarchy without chrome

  • Type + hairlines carry hierarchy. No card surfaces except the user's chat bubble (the one place "you said this" demands weight).
  • Amber = signal. Used for one thing per screen at a time. If the orb is amber, nothing else can be.
  • Calistoga for a human voice (on-device: Montserrat Bold 48 carries equivalent weight). Inter for body (on-device: Montserrat Regular 20). JetBrains Mono for machine output (on-device: Montserrat 14 uppercase + letter-spacing).
  • One hero, full stop. Every screen has exactly one thing that moves or glows. Never two.

LVGL realism

  • Orb = 1 lv_obj with gradient bg + shadow_width=60 spread=20 opa=95 · breathing via lv_anim on shadow_opa.
  • Ripples = 3 concentric lv_obj with transparent bg, amber border, alpha+scale anim staggered.
  • Stream gutter = 1 thin rect (1×560) + 3 dots + 3 labels = 7 obj. Visible only during active agent runs.
  • All spacing uses DPI_SCALE(px) from config.h. This whole system ports.

Trust + safety

  • Privacy before pretty. Mic mute + wake-word opt-in + conversation-storage consent before any feature shows.
  • Quiet hours are a first-class setting, not buried. A voice device that won't shut up is a liability.
  • Mode is visible on the orb always. You never lose track of what's listening to you.
  • Error screens speak plainly. "My brain is offline" beats "Error: connection to 192.168.x.x failed (errno 113)".
