widget icons · v1 · the vocabulary

Sixteen icons. Drawn in code.

Every widget can carry an icon slot — a single 48 px glyph that sits top-right, identifying the skill without words. Icons are drawn server-side as pure SVG paths using primitives — circle, line, rect, arc. New icon = 5 lines of Python, shipped as part of the skill manifest. Tab5 renders them via LVGL canvas draw calls.

◦ rules

All same family.

Stroke 1.8 px.

Every icon in the library uses the same stroke width. Rendered in LVGL via lv_line and lv_arc at 2 px. Mixing thicknesses breaks the rhythm.

48 px canvas.

All icons authored on a 48 × 48 grid. Rendered at any device-appropriate size via scale transform. Safe area: 4 px from edges.

No fill by default.

Outline only. Filled shapes (eyes, dots) are the exception. Keeps icons visually light against dark backgrounds.

One color slot.

All primitives inherit the icon's color. Default: #D8D8DC at 0.78 opacity. Tone can shift via widget: success → #22C55E, alert → #EF4444.

Rounded joins.

stroke-linecap: round; stroke-linejoin: round;. Hard miters look brittle on a voice-first device.

No emoji. Ever.

Emoji are font-dependent, inconsistent across platforms, can't be themed. Every icon is drawn from primitives by us.
