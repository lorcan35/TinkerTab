TinkerTab · Widget Platform · V1

The screen becomes an extension of the brain.

Today, every new feature on Tab5 means editing C, recompiling, flashing. 196 skills live on Dragon and none of them show up on the device. The platform is storefront without goods.

The Widget Platform fixes that. Skills emit state. Tab5 renders it through six typed widgets. Write a Python file, pick a widget, done. No firmware flash. Every future idea ships in minutes, not weeks.

01 · Architecture

Three layers, strict contracts between them.

Skills live on the brain. Tab5 owns rendering. The protocol between them is a small typed vocabulary. Any device that implements the vocabulary gets every skill for free — Tab5, Waveshare round display, a bare ESP32-S3 with a 128×64 OLED. The product stops being the hardware and becomes the design language + the skill registry.

02 · Vocabulary

Six widgets. Not twenty.

Every skill fits into one of these shapes. Unlimited data inside each, but the interaction contract is fixed. That's how skills stay portable across devices and how the UI never becomes a novice-designed mess.

◦ LIVE
One thing, now.

Owns the home hero slot. Replaces the orb's idle color + poem. Exactly one live widget at a time — brain picks by priority weight.

title · body · icon
tone calm|active|approaching|urgent|done
progress 0..1
action {label,event}
priority 0..100
timer · drafting · heartbeat · routing
◦ CARD
A moment in conversation.

Lands in chat or the activity feed. Short-form. Optional single amber action. For completions, alerts, notifications with optional reply.

title · body · image_url?
tone info|success|alert
action {label,event}?
timer done · email drafted · meeting in 10
◦ LIST
Pick one.

Scrollable row list, each row with title + optional subtitle. Tap-to-select emits the item's id. For "which recipe?" "which song?" "which room?"

title
items[] {id,title,subtitle?}
on_select event name
pick a recipe · choose a room · route to …
◦ CHART
A number with a shape.

Sparkline, bar, or single gauge. Dragon sends normalized points; Tab5 picks the best rendering for its screen. Low-capability devices degrade to "avg: 42, trending up".

title · series[]
kind spark|bar|gauge
unit? · range?
spend this week · focus minutes · mood arc
◦ MEDIA
A picture with a caption.

Image or generated visual + label. Reuses the existing rich-media pipeline (Pygments code blocks, Pillow tables → JPEG). The escape hatch for anything the typed widgets can't express.

url · alt
caption? · w/h?
action? {label,event}
generated chart · rendered code · photo
◦ PROMPT
Ask then act.

One question with a typed expected response — number, text, choice. Keyboard or voice answers both honored. For "how many reps?", "which person?", "confirm send?"

question · input
{kind,placeholder?,choices?}
on_answer event name
how many? · confirm? · which dog?
03 · Author experience

A whole skill, under 80 lines.

This is the entire author-facing surface. A Python file + a short YAML manifest. No layout syntax, no CSS, no LVGL. The manifest lists voice triggers and which widget slots the skill uses. The Python emits state into those slots. Everything else is the platform's problem.

skills/timesense/ manifest.yml

Metadata only. Voice triggers, which widget surfaces, default icon, priority weight. No layout instructions — that's the renderer's job.

skill_id: timesense.pomodoro name: Time Sense description: AI-first timer voice_triggers: - "set a timer for {min:int} minutes" - "pomodoro" - "deep work for {min:int}" icon_default: briefcase surfaces: - widget: live priority: 80 when: running - widget: card when: done

skills/timesense/ skill.py

Pure Python. Emits widget state. Dragon handles WS transport, voice trigger matching, and capability-aware routing.

# timesense.py — the whole thing. from openclaw import Skill, Surface class Timesense(Skill): def start(self, minutes: int): self.remaining = minutes * 60 self.surface.live( title="Deep work", body=f"{minutes}:00 remaining", icon="briefcase", tone="calm", progress=0, action=("PAUSE", "ts.pause"), priority=80, ) self.schedule_tick(1) # tick every 1s def on_tick(self): self.remaining -= 1 pct = 1 - self.remaining / self.total tone = "urgent" if pct > .95 else \ "approaching" if pct > .9 else \ "active" self.surface.live_update( body=fmt(self.remaining), tone=tone, progress=pct, ) if self.remaining <= 0: self.surface.live_clear() self.surface.chat_card( title="Done", body=f"{self.total//60} min · well run", tone="success", )
04 · Rules we keep

Opinionated on purpose.

The platform enforces taste by vocabulary. Authors can't make ugly layouts because there's nothing to lay out. These are the non-negotiables for v1.

1
One live, full stop.
Exactly one widget owns the orb at a time. Brain resolves priority. Others queue.
2
Amber is the only accent.
One signal color per moment. Tone modifiers use existing status red / emerald / rose — never new hues.
3
No layout from skills.
Manifest declares surfaces; never coordinates. Renderer owns everything visual.
4
Renderer degrades gracefully.
A chart on an OLED becomes "avg 42 ↑". A prompt on a keyboard-less device becomes voice-only.
5
Typography carries meaning.
Letter-spaced caption = state. Serif display = headline. Amber italic = signal. Never color alone.
6
Actions are always single.
Widgets expose at most one primary action. Secondary actions surface via long-press.
7
Idle has no widget.
No skill forced onto an empty home. Calm orb is the default, always.
8
Skills never block.
Async everything. Tab5 UI stays interactive while skills run on Dragon.
05 · What this is not

No app store. No layout engine. No JavaScript.

Every feature we explicitly cut. Skills are not apps. Widgets are not layouts. The vocabulary is the product.

No layout YAML / HTML.

Skill authors never write row, column, label x:12 y:42. The vocabulary is the layout. This is the Garmin watchface trap; we avoid it.

No interpreter on Tab5.

No JS, no Lua, no sandboxed code. Skills run on the brain in Python. Tab5 only renders JSON state. Security model is trivial.

No app install flow.

Skills sync from Dragon's registry. Install a skill = git pull on the brain. No firmware flash, no Tab5 restart, no per-device distribution.

No third-party UI chrome.

Every widget rendering obeys v5 tokens. Skills can't override colors, fonts, or spacing. Taste is centrally enforced.
